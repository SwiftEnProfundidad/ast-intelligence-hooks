# Backend AST Rules

Esta sección documentará todas las reglas AST aplicadas a proyectos **Backend** (TypeScript / NestJS).

## Estructura

Cada regla seguirá este formato:

- **Id**: `backend.<dominio>.<nombre>`
- **Severidad**: `critical | high | medium | low | info`
- **Descripción**: Qué problema detecta y por qué importa.
- **Ejemplo que viola la regla**: Fragmento de código problemático.
- **Ejemplo correcto**: Cómo debería verse el código.
- **Cómo desactivarla o ajustar severidad**: Configuración recomendada.
